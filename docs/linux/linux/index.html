1. 固定IP地址
#

/etc/dhcpd.conf 文件

interface wlan0
static ip_address=192.168.1.110/24  # ip和子网掩码（可能被占用，就需要更改）
static routers=192.168.1.1          # 网关
static domain_name_servers=192.168.1.1  # DNS地址

/etc/resolv.conf 文件 (DNS配置文件会自动加入)

# Generated by resolvconf
nameserver 10.11.248.114
nameserver 10.11.248.115
nameserver 192.168.1.1

2.开机启动
#

方法一、修改 /etc/rc.local 文件

sudo nohup python -u /home/pi/Downloads/weather_pi/weather.py >> /home/pi/Downloads/weather_pi/result.txt 2>&1 &

方法二、在 /home/pi/.bashrc 文件的末尾添加启动命令文本。

echo Running at boot 
sudo python /home/pi/sample.py

方法三、把应用程序加入到 /etc/init.d 目录

需要加入启动配置信息 https://wiki.debian.org/LSBInitScripts

# /etc/init.d/sample.py
### BEGIN INIT INFO
# Provides:          sample.py
# Required-Start:    $remote_fs $syslog
# Required-Stop:     $remote_fs $syslog
# Default-Start:     2 3 4 5
# Default-Stop:      0 1 6
# Short-Description: Start daemon at boot time
# Description:       Enable service provided by daemon.
### END INIT INFO

确保这个脚本具有执行权限。

sudo chmod +x sample.py
sudo update-rc.d sample.py defaults

方法四、配置 systemd 文件 /lib/systemd/system/sample.service

权限需要设置成 sudo chmod 644

添加下面的内容：

  • 名为 Sample Service 的服务，我们希望在多用户环境下依然有效启动一次。
  • Type 设置为 idle 是为了确保其他一切都已加载之后再执行
 [Unit]
 Description=My Sample Service
 After=multi-user.target

 [Service]
 Type=idle
 ExecStart=/usr/bin/python /home/pi/sample.py

 [Install]
 WantedBy=multi-user.target

方法五、crontab 文件

指令

service cron start
service cron restart
service cron stop
service cron status
systemctl is-enabled cron.service  --- 查看crond是否为开机自启动
systemctl enable cron.service  --- 将服务设置为开启启动
systemctl disable cron.service  --- 关闭服务开机自启动

内容

SHELL=/bin/bash
PATH=/sbin:/bin:/usr/sbin:/usr/bin
MAILTO=root

# 更多细节 man 4 crontabs

# 计划任务定义的例子:
# .---------------- 分 (0 - 59)
# |  .------------- 时 (0 - 23)
# |  |  .---------- 日 (1 - 31)
# |  |  |  .------- 月 (1 - 12)
# |  |  |  |  .---- 星期 (0 - 7) (星期日可为0或7)
# |  |  |  |  |
# *  *  *  *  * 执行的命令
* * * * * date >> /time.txt 2>&1 % stdin input

固定值 		1
列表值 		2,12,22
连续范围值	   1-7
步长值			1-24/3
第一个%表示标准输入的开始,其余%表示换行符

定时语句解析工具 https://crontab.guru/

易错例子 0 3 * * * command 每天三点执行

0 0 1-7 * * command 每月的1-7每天零时执行

0 0 * * 1 command 每星期的星期一零时执行

0 0 1-7 * 1 command 每月的1-7和每星期一零时执行

日志文件位置 /var/log/cron

  21. scp 用于 Linux 之间复制文件和目录

    是 secure copy 的缩写, scp 是 linux 系统下基于 ssh 登陆进行安全的远程文件拷贝命令。scp 是加密的，rpc是不加密的，scp 是 rcp 的加强版

    scp localfile username@ip:folder	# 复制文件到远程文件夹，保留名称，只需要输入密码
scp localfile username@ip:file		# 复制文件到远程，修改名称，只需输入密码
scp localfile ip:folder				# 复制文件到远程文件夹，保留名称，需要输入用户名和密码
scp localfile ip:file				# 复制文件到远程，修改名称，需要输入用户名和密码

scp -r localfolder username@ip:folder	# 复制本地文件夹到远程目录下，只需要输入密码
scp -r localfolder ip:folder			# 复制本地文件夹到远程目录下，需要输入用户名和密码

  22. awk 文本分析工具

    awk [参数] [处理内容] [操作对象]
查看修改代码行数：git log --author="i_matijun" --oneline --shortstat | grep "files changed" | awk 'BEGIN {plus=0; minus=0} {plus+=$4; minus+=$6} END {print "增加行数：" plus "   减少行数：" minus}'
查询每个人提交次数：git shortlog -s -n

    内建变量：

    • NF 当前处理行的字段个数【列数】

    • NR 当前处理行的行号

    • $0 表示当前处理的整行内容

    • $n 表示当前处理行的第 n 个字段

    • FILENAME 被处理的文件名称

    • RS 行分隔符，预设值为 \n

    参数示例：

    • -F ':| ' 使用冒号和空格分隔一行
处理内容：一个或多个 `pattern {action}` 若 `pattern` 为非 0 值，则执行 `action`

- `'$15 > 60 {print $12, $15}'` 分隔后第十五列【】大于60 时，格式化输出12列和15列的值
- `'NR==1,NR==3 {print}'` 输出 1 至 3 行内容
- `'NR==1;NR==3 {print}'` 输出 1 和 3 行内容
- `'(NR>=1)&&(NR<=3) {print}'` 输出 1 至 3 行内容
- `'BEGIN{sum=0;n=0} {n+=1;sum+=$15} END{print sum,n,sum/n}'` 求第 15 列 平均值
- `'/[^CT]all/'` 匹配正则表达式 \[^CT]all
- `'BEGIN { for (i = 1; i <= 5; ++i) print i }'` 使用循环
- `'+$1 { printf("%10s | %4d\n",  $3, $1) }'` 第一列非 0 则使用 printf 函数格式化输出。模式中的一元加号强制在数字上下文中对 $1 求值。

#

  1. tar

    tar -czvf test.tar.gz file1 file2 … 将文件打包为test.tar.gz

    -z 使用gzip压缩

    -j 使用bzip2压缩

    -v 显示指令执行过程

    -c 创建包

    -f 指定备份文件（必须在最后）

    -t 列出包的内容

    • tar tf test.tar.gz

    -x 解压

    -C <目的目录> 切换到指定目录

    • tar zxf test.tar.gz positionA/

  2. gzip 对打包的文件压缩、解压缩

    -r 压缩.gz文件

    -d 解压

  3. zip -r 压缩

    unzip -d 解压

    压缩率：zip < gzip < bzip2

    通用性：zip > gzip > bzip2

  4. which 查看命令的位置

  5. su 切换用户

  6. passwd UserName newPassword 修改密码

  7. who 查看当前登录的用户

  8. reboot 重启（无需权限）

  9. shutdown (root权限)

    -r now 重新启动

    -h now 立即关机

    -h +10 十分钟后关机

    -h 20:20 定时关机

  10. chmod 权限设置

    rwx:读、写、可执行

    或者4210：读、写、可执行、无权限

    chmod 777 fileA 将全部的权限赋给三组用户

    d表示是一个目录

    前三个：所有者的权限 u

    中三个：所属组的权限 g

    后三个：其他用户的权限 o

    a 所有用户

    + 添加权限

    - 删除权限

    = 设置权限

    chmod u+r fileA 将文件A的读权限添加给u

  11. sudo gpasswd -a <你的用户名> docker 将用户添加到docker组

    添加后重启会话, 这一步是必须的，否则因为 groups 命令获取到的是缓存的组信息，刚添加的组信息未能生效

性能分析指令
#

  1. top [-] [d delay] [q] [c] [S] [s] [i] [n] [b]

    • -d <秒数>：指定 top 命令的刷新时间间隔，单位为秒。
    • -n <次数>：指定 top 命令运行的次数后自动退出。
    • -p <进程ID>：仅显示指定进程ID的信息。
    • -u <用户名>：仅显示指定用户名的进程信息。
    • -H：在进程信息中显示线程详细信息。
    • -i：不显示闲置（idle）或无用的进程。
    • -b：以批处理（batch）模式运行，直接将结果输出到文件。
    • -c：显示完整的命令行而不截断。
    • -S：累计显示进程的 CPU 使用时间。

    面板数据：

    %us：表示用户空间程序的cpu使用率（没有通过nice调度）

    %sy：表示系统空间的cpu使用率，主要是内核程序。

    %ni：表示用户空间且通过nice调度过的程序的cpu使用率。

    %id：空闲cpu

    %wa：cpu运行时在等待io的时间

    %hi：cpu处理硬中断的数量

    %si：cpu处理软中断的数量

    %st：被虚拟机偷走的cpu

    PR（优先级）：进程的优先级

    NI（Nice值）：进程的优先级调整值。

    VIRT（虚拟内存）：进程使用的虚拟内存大小。

    RES（常驻内存）：进程实际使用的物理内存大小。

    SHR（共享内存）：进程共享的内存大小。

    S：运行状态。

       - R 运行
   - S 睡眠
   - I 空闲
   - D 磁盘休眠 （不可中断睡眠状态）
   - T 停止
   - Z 僵尸
   - X 死亡（几乎捕捉不到）

    TIME+：进程的累计 CPU 时间。

    交互：

    q 退出top界面 ?、h 进入帮助界面 m 切换内存信息显示 t 切换cpu状态显示 l 触发负载均值显示 1 显示cpu核数信息 L 定位关键字 n、# 显示最大任务数量 u、U 过滤所属用户的进程 R 根据进程号(PID)排序(顺序或逆序) H 触发线程显示 C 显示当前行、列所在总行列数的位置 up、down、left、right 上下移动行，左右移动列，通常配合C使用 home、end 跳转至第一行、最后一行 k 根据PID杀死对应的进程 Z、z 颜色设置，触发颜色 d、s 设置top刷新周期，默认3秒刷新一次 f、F 进入设置界面，可设置top显示的项目和排序规则

  2. netstat

    -a 详细的网络状况

    -nu 当前户籍UDP连接状况

    -apu UDP端口号的使用情况

    -i 网卡列表

  5. free 显示内存状态

    -s<间隔秒数> 　持续观察内存使用状况
